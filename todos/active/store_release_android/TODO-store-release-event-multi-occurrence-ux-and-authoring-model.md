## Constraint Notes

- **Active constraint:** `BUG-SR-RECUT-03`
- **Constraint rationale:** The public/read side shows persisted programação, but the tenant-admin root-level programação editor for single-occurrence events does not hydrate from the first occurrence on edit, making persisted data appear missing and risking accidental overwrite.
- **Clearance path:** Hydrate root-level Programação from the first occurrence when and only when there is exactly one occurrence; hide the root section for multi-occurrence events; prove save/reopen/public readback and single-to-multi absorption with mutation/navigation evidence.
- **Owner / source:** Store Release orchestrator final runtime validation.
- **Active blocker:** `None`
- **Last confirmed truth:** Home/public discovery remains occurrence-first and list cards remain occurrence-only. Those occurrence cards/chips use occurrence-owned related profiles only; event-shared profiles do not leak into the occurrence card. Public detail related profiles aggregate event + all occurrences deduplicated. The single-occurrence tenant-admin root Programação editor is reopened because it can fail to hydrate from persisted first-occurrence programação on edit.
- **2026-04-23 blocker note:** Runtime/code-path investigation identified a domain-consistency risk outside the visible UI slice: the scheduled `events:occurrences:reconcile` path could rebuild occurrences from an incomplete shape and therefore degrade occurrence-scoped `programming_items` and own related-profile data. That blocker was resolved in `TODO-store-release-event-occurrence-transactional-consistency-and-reconcile-removal.md`, and SR-D2 runtime acceptance no longer depends on any background reconcile path.
- **2026-04-23 contract supersession note:** Earlier DOD/evidence rows that described programação participants auto-linking directly into the event-level related-profile set are historical only. Recut implementation and runtime evidence against the occurrence-first participant-sourcing model before this TODO is considered fully reconciled again.
- **2026-04-23 visual polish supersession note:** Earlier evidence rows that validated the old `Programação` date chips (`Atual` + horário), title-from-profile fallback, location-as-chip rendering, or the old `Como Chegar` complementary-card labeling are historical only. Recut DOD/VAL/NAV evidence against the approved visual contract below before this TODO is considered fully reconciled again.
