## Wave 2 Addendum (2026-04-30)
- **Addendum ID:** `W2-OCC-SESSION-CONTEXT`
- **Why it stays in this plan:** the behavior belongs to the same invite share-code continuation story, uses the same `occurrence_id` target identity, and does not introduce a new approval conversation, remote entity, or independent product slice.
- **Approved Store Release contract:** after `GET /invites/share/{code}` succeeds, Flutter may hold the preview context in app/session memory keyed by `share_code + occurrence_id`. Event detail and invite surfaces may consume that projection so the user sees the pending invite effect before explicit acceptance.
- **Persistence boundary:** no persisted local storage, no remote anonymous invite-intent entity, and no `invite_edge` creation before explicit acceptance.
- **Canonical app accept path:** app/native explicit accept remains `POST /invites/share/{code}/accept` after authenticated identity is available; anonymous app/web accept attempts must be blocked before backend mutation or rejected with `401 auth_required`, and anonymous web accept/CTA actions must hand off to app promotion with `/invite?code=...` without calling backend accept.
- **VNext split:** remote anonymous invite-intent persistence, anonymous identity merge, multi-session recovery, metrics, dedupe beyond share-code contract, and promotion of intent to invite edge are deferred out of this plan.
- **Completion gate:** focused Flutter preview/session evidence, module documentation sync, web build, and source-owned Playwright runtime mutation proof are complete for the web-boundary portion of the addendum; `W2-OCC` cannot return to renewed promotion review until authenticated app/native share-code accept evidence is refreshed.
