# T5 Audit Package Round 01 - Agenda Card Polish And Occurrence Taxonomy

## Scope
This package covers the approved T5 Store Release slice:

- Public agenda/card polish in Flutter: Account Profile chip compression as `e mais X`, stable card trailing action slot, Home Agenda invite/radius chrome, and `às` for explicit time ranges.
- Backend Events contract changes: optional `programming_items[].end_time`, occurrence-owned taxonomy overrides, validation against the parent event category, and effective occurrence taxonomy filtering.
- Flutter admin/public consumers for those backend contracts.
- Replacement of the app `BooraIcons` font asset with the uploaded 55-icon font and tenant-admin icon picker coverage.
- Tenant-public web desktop-frame exception for the Map route family only.

Out of scope: contacts/friends materialization, invite lifecycle fixes, favorite refresh, deep-link host/domain rules, public/private discovery rules, and production promotion.

## Implementation Summary
Flutter changed public card/schedule surfaces, admin event authoring surfaces, DTO/domain mapping, shared icon catalogs, and the public web desktop frame. The Map route family (`CityMapRoute` and `PoiDetailsRoute`) now bypasses the tenant-public max-width frame, while other tenant-public routes remain framed.

Laravel Events now accepts, validates, persists, and projects optional programming item `end_time`; accepts occurrence taxonomy overrides through `taxonomy_terms`; stores occurrence-owned taxonomy terms as `own_taxonomy_terms`; and uses effective occurrence taxonomy for public filtering, where occurrence terms replace event terms when present and event terms remain fallback when absent.

Boora font source artifacts were consolidated into the existing app font family and source class:

- `flutter-app/assets/fonts/BooraIcons.ttf`
- `flutter-app/assets/fonts/boora_icons_configs/config.json`
- `flutter-app/lib/application/icons/boora_icons.dart`
- `flutter-app/lib/presentation/shared/icons/map_marker_icon_catalog.dart`

Legacy/default storage keys remain resolvable through catalog aliases.
