# Bounded Package: Account Profile Queryability and Public Navigation Contract

## Scope
This package is bounded to `foundation_documentation/todos/active/v0.2.0+8/TODO-v0.2.0+8-account-profile-queryability-navigation-contract.md`.

It covers only the queryability/listability and public-detail navigability contract for Account Profiles, the canonical Laravel query gateway/guard, the explicit backend payload contract consumed by Flutter, and the user-visible public/admin/browser behavior needed to stop non-queryable or non-navigable profiles from leaking into public navigation and selectors.

It does **not** include unrelated work already present on the branch, including event invite/share polishing, crop UX, hero polish, taxonomy refactor, or previously-authored web navigation specs outside this TODO.

## Problem Reproduced
Manual validation found a public Event related-profile card under tab `Outro Grupo` that appeared non-linkable but still navigated to `/parceiro/:slug`, leading to `Algo deu errado`.

Concrete route from the TODO:

- `https://guarappari.belluga.space/agenda/evento/pw-event-share-boundary-store-release-5?occurrence=6a203315dcd27475bd0ce16d`

## Canonical Decisions Under Audit
- `is_queryable` and public direct-detail navigability are independent capabilities.
- Non-queryable profiles must not leak into selectors, public groups, discovery, map/filter options, or favorite/public surfaces.
- Public detail navigation must not be inferred from `slug`; the backend must project explicit navigability.
- Query enforcement must be centralized in canonical Laravel gateway/service paths and protected by deterministic guardrails with allowlist audit output.
