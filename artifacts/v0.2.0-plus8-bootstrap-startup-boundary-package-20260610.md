## Frozen Audit Goal

Audit whether the current local implementation:

1. establishes one canonical tenant-public identity-readiness boundary;
2. removes broad auth bootstrap side effects from protected public request helpers;
3. fixes the first permission-granted web map entry by cause-root ownership rather than mutable runtime hacks;
4. preserves the absorbed first direct public-route startup rule: initial public entry stays on the public surface unless there is a real invite/deferred override or an installed-app handoff succeeds;
5. avoids gambiarra / hidden compatibility shims in the final architecture.

## Implementation Intent

- Protected tenant-public request helpers must no longer call broad `AuthRepository.init()` just to obtain an anonymous bearer.
- Web location permission grant must not leave the permission screen until the runtime has a canonical path to finish location bootstrap; if the same SPA document cannot reliably read geolocation immediately after grant, the fix must be document/startup owned instead of a mutable global bypass.
- Anonymous web policy remains `anonymous-authenticated`; raw unauthenticated public reads are not allowed.

## Canonical Authorization and Cutover Bounds

- The governing TODO now explicitly authorizes the chosen web grant-completion owner:
  - `D-07`: same-origin fresh-document reentry is the canonical web boundary for permission-granted map entry when the pre-grant SPA document cannot yet read geolocation reliably.
  - `Decision Baseline D-06`: future simplification is allowed only if it preserves the same route-owned/bootstrap-owned ownership and the same first-request guarantees.
- This package is therefore not claiming a hidden compatibility bridge. It is claiming an explicit, bounded canonical owner decision promoted into the TODO before audit closure.
- The previously suspicious mutable runtime bypass was removed:
  - deleted `flutter-app/lib/application/router/guards/location_permission_gate_runtime.dart`
- The surviving web handoff is narrow and route-owned:
  - `tenant_public_map_entry_flow.dart` decides the permission outcome;
  - `location_permission_granted_document_reentry_web.dart` performs same-origin `window.location.replace(...)`;
  - there is no mutable singleton carrying permission state across flows.
