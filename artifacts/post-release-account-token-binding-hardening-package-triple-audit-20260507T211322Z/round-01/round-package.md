## Real Drift Fixture

- Account-scoped abilities were copied into Sanctum tokens without being strongly bound to the currently selected account context.
- A user belonging to multiple accounts with different role/ability levels could risk satisfying Account B routes with Account A token abilities.
- Drift fixtures are the account route surfaces plus `CheckUserAccess`, `AccountAuthenticationService`, `TenantScopedAccessTokenService`, and `AccountUser::tokenCan()` authorization semantics.

## Frozen Rule Set

### Violated Rule

- Account-scoped routes authorized too much from token ability strings and not enough from current account context.

### Replacement Canonical Rule

- Account-user bearer tokens are stamped with `account_id` at issue/stamp time.
- Account-scoped routes reject bearer tokens with missing or mismatched `account_id` against `Account::current()`.
- Sanctum token abilities are only a ceiling.
- The Sanctum ceiling is evaluated with account-workspace wildcard semantics before live permission revalidation: literal `*`, exact abilities, and resource wildcards such as `account-users:*`, `events:*`, and `push-messages:*` can satisfy the ceiling only for the same account-bound request.
- `AccountUser::tokenCan()` revalidates live current-account role permissions only when `CheckUserAccess` has established account-scoped auth context through the `account` middleware.
- Account-scoped token issuance fails closed when account-scoped abilities are requested without a resolvable account context; unbound reusable account-workspace bearer tokens are not valid output.
- Direct `AccountUser::createToken()` account-scoped issuance fails closed unless it runs inside a validated issuer context opened by `TenantScopedAccessTokenService`.
- Account-prefixed package routes that depend on `Account::current()` must use the `account` middleware or an equivalent fail-closed binding guard; this package explicitly includes the push message `data` and `actions` routes.
- Account A permissions cannot satisfy Account B routes through copied token abilities.
